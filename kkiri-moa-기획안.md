## 1. 개요 · 목적

- **학습 목적**: 사물을 **모양 · 색깔 · 크기**의 속성으로 **분류**하는 경험을 통해, 유아가 생활 속에서 수학적 사고(분류·비교)를 재미있게 익힌다.
- **대상**: 유아 ~ 초등 저학년.
- **핵심 개념**: 같은 속성을 가진 것끼리 모으기(분류). 동일한 사물도 **기준을 바꾸면 다르게 묶인다**는 것을 체험.
- **상호작용 유형**: 드래그 & 드롭. 같은 것끼리 **세로 한 줄(칼럼)**로 모은다.

## 2. 전체 구조 (총 3단계)

| 단계 | 분류 기준 | 칼럼(줄) | 크기 |
|---|---|---|---|
| **1단계** | 모양 | 세모 · 네모 · 동그라미 · 하트 (4) | 모두 동일 |
| **2단계** | 색깔 | 빨 · 노 · 초 · 파 (4) | 모두 동일 |
| **3단계** | 크기 | 꽃나라 · 별나라 · 물나라 (선물 크기) | 크기 변수 |

- 1·2단계는 **같은 오브젝트 12개**를 공유한다(아래 §4). 1단계는 모양으로, 2단계는 색으로 다시 분류 → "같은 것을 기준 바꿔 분류" 경험.
- 3단계는 크기·전용 에셋(산타·선물·나라 표지)이 필요해 별도 구성(§7).

## 3. 화면 흐름

```

   ▲                                          │ 모든 줄 채움
   │                                          ▼
게임오버 ◀──── 3단계(예정) ◀──── 2단계 ◀──── (단계 완성: confetti + 효과음)
```

- 마지막 단계 후: 게임오버 카드(축하 + "다시 하기" 버튼).

## 4. 콘텐츠 데이터 (1·2단계 공용 오브젝트 12개)

각 오브젝트는 `{shape, color}`를 가지며 **크기 동일**. 모양 4종 × 색 4종을 **균형 설계**(각 모양 3개, 각 색 3개)하여 어느 기준으로 나눠도 칼럼당 3개로 떨어진다.

| # | 모양 | 색 | | # | 모양 | 색 | | # | 모양 | 색 |
|---|---|---|---|---|---|---|---|---|---|---|
| 1 | 세모 | 빨 | | 5 | 세모 | 노 | | 9  | 세모 | 초 |
| 2 | 네모 | 노 | | 6 | 네모 | 초 | | 10 | 네모 | 파 |
| 3 | 동그라미 | 초 | | 7 | 동그라미 | 파 | | 11 | 동그라미 | 빨 |
| 4 | 하트 | 파 | | 8 | 하트 | 빨 | | 12 | 하트 | 노 |

- 색 HEX: 빨 `#E8503A` · 노 `#F4C430` · 초 `#4CAF50` · 파 `#3E8FD0`
- 1단계 칼럼: 세모/네모/동그라미/하트(각 3개) · 2단계 칼럼: 빨/노/초/파(각 3개)

## 5. 게임 로직 (1·2단계)

1. 하단 **풀(pool)**에 12개 오브젝트가 흩어져 있다(드래그 시 살짝 커지며 집어 올림 + pickup 효과음).
2. 위쪽 **세로 칼럼 4개** 중 기준에 맞는 칼럼에 끌어다 놓는다.
   - **정답**: 칼럼 안에 위→아래로 **세로 스택**되어 고정(snap 효과음 + 칭찬 말풍선 + 칼럼 하이라이트).
   - **오답**: 흔들리며 제자리로 복귀(오답 말풍선).
   - 칼럼 밖: 제자리로 복귀.
3. 12개를 모두 올바른 칼럼에 넣으면 **단계 완성** → 다음 단계.

## 6. UI · 비주얼 가이드

- 캔버스 1024×700, 따뜻한 파스텔 그라데이션 배경 + 부유하는 점.
- 상단 헤더: 배지("분류") + 제목("끼리끼리 모아") + 단계·기준 칩 + 단계 진행 점.
- 칼럼: 둥근 모서리 점선 레인 + 헤더 밴드(아이콘 + 라벨). 모양 칼럼은 도형 아이콘, 색 칼럼은 색 스와치.
- 오브젝트: 그림자·광택의 **입체 종이타일** 느낌(그라데이션 + 외곽선 + 하이라이트). 하트는 파라메트릭 곡선으로 렌더.
- 가이드 캐릭터 + 말풍선: 우측 하단 거터. 상태별 말풍선(기본/정답/오답/단계완성).
- gsap(집기·스냅·복귀 트윈), canvas-confetti(단계완성·게임오버).

## 7. 3단계 설계 (예정 · 미구현)

가이드: *"꽃나라/별나라/물나라가 구분되어 있는 표지에 따라 다양한 크기 선물 분류 (산타클로스 및 선물 모양)"*

- **기준**: 크기. 선물 상자가 크기별(큰/중간/작은)로 흩어져 있고, 세 나라 표지(꽃나라/별나라/물나라)에 맞는 크기로 배달.
- **매핑(안)**: 예) 큰 선물 → 별나라, 중간 → 꽃나라, 작은 → 물나라 (표지에 크기 가이드 표기). 정확한 대응은 영상/가이드 추가 확인 필요.
- **에셋 필요**: 산타클로스, 선물 상자(대/중/소), 세 나라 표지/배경.
- 1·2단계의 칼럼 메커니즘을 재사용하되 칼럼 라벨을 "나라"로, 판정 기준을 크기로 교체하면 구현 가능.

## 8. 에셋 계획

| 종류 | 가이드 요구 | 현재 |
|---|---|---|
| 배경음 | 차분한 배경음 | ✅ `assets/bg_music.mp3`(기존 재사용) |
| 효과음 | 더빙 편집 or 타자기 소리 | ⏳ pickup/place/correct(기존)로 대체, 더빙·타자기 미적용 |
| 캐릭터 | 화면별 **문어/호랑이** | ⏳ 공용 가이드 캐릭터 placeholder |
| 배경 | 문어=바다속 책, 호랑이=깔끔 | ⏳ 공용 파스텔 배경 |
| 도형 | 3D 종이타일/입체 | ⏳ 캔버스 의사-3D(그림자·광택) |
| 3단계 | 산타·선물·나라 표지 | ❌ 없음 |

## 9. 기술 구조

- 엔진 보일러플레이트(참고앱 `_ref/5/`에서 재사용): `index.html`, `main.ts`(스케일링·부모 iframe 메시지), `appHelper.ts`(데이터 로드·좌표 변환·캡처), `style.css`, `tsconfig.json`.
- 게임: `app.ts` / 콘텐츠: `data.json`(`appData`·`textData`·`assetList`).
- 빌드: esbuild로 `main.ts`→`main.js` 단일 번들(gsap/howler/confetti/html-to-image 등 external). 명령은 메모리 `kkiri-moa-build-verify` 참고.
- 검증: 헤드리스 Chrome(playwright)로 단계 자동 풀이 + 콘솔 에러 0건 확인. `?debug` 시 `window.__kkiri.snapshot()` 읽기전용 훅.
